# RoboClaw

**RoboClaw** 是一个面向具身智能助手的开源共创框架。

它受到 OpenClaw 启发，但它不是“给机器人套一层 OpenClaw”。

RoboClaw 以四层结构组织系统：

- `助手层`：负责用户、会话、智能体编排、工具路由和远程接入
- `具身层`：负责本体建模、空间建联、能力抽象、熟悉校准、训练辅助、急停判断与恢复支持
- `执行层`：以 ROS2 作为执行中间层，负责控制器、消息通道、服务调用、动作执行、安全监督和状态回传
- `载体层`：负责连接仿真环境与真实机器人，承接部署、验证和回传

## 为什么做 RoboClaw

具身智能需要的不只是更强的模型，还需要一个能把本体、传感器、动作、执行和部署组织起来的可复用架构。

RoboClaw 试图避免三类常见错误：

1. 把系统做成机器人专用胶水代码
2. 把原始 ROS 接口当成最终抽象
3. 只靠 prompt 处理不同本体的差异

## 系统路径

`goal -> planner -> semantic skill -> supervisor -> execution layer -> carrier layer`

RoboClaw 更偏向语义动作和受约束监督，而不是让模型直接输出低层电机控制。

## 原则

- 显式接口优先于隐藏耦合
- 语义动作优先于原始关节控制
- supervisor 优先于不受约束的模型行为
- 框架质量优先于 demo 式捷径

## 当前状态

RoboClaw 仍处于早期。方向已经明确，但实现范围仍然刻意保持收敛。

当前没有宣称已经完成：

- 完整具身运行时
- 稳定的多机器人抽象
- 完整部署与验证栈
